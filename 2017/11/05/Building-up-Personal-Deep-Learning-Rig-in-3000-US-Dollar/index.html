先附上沒插卡的 ASUS STRIX Z270F GAMING 主機板玉照一張

硬體規格

  • CPU: Intel Core I7-7700K / 4.2G
  • 風扇: INTEL BXTS15A
  • M/B: ASUS STRIX Z270F GAMING
  • RAM: Kingston 16G DDR4 2400 * 4 = 64G
  • SSD: Kingston 240G M.2 2280
  • VGA: ASUS TURBO-GTX1080TI-11G * 2
  • DVDRW: SATA
  • POWER: 1000W, Cooler Master V1000 80Plus 金牌
  • 殼: Cooler Master CM6900III
  • HD: WD (紅標?) 3TB

(2017.11.07 更新) Tensorflow 有許多裝法，本篇是 f26 + docker ，其他裝法如果後續有嘗試我會在下面補充

  • Ubuntu 16.04 + Virtualenv
  • Fedora 26 + Docker (這篇)

Overview

我的目標是 硬體組好、 Hello Tensorflow 跑起來，host OS 會使用 fedora，官方教學只有 Ubuntu 的，大概是覺得 Ubnuntu 是最多人用的發行版，但還是可以參考一下，至於為什麼用 fedora ，純粹就是個人偏好 RPM

tf 用 python 環境安裝方式主要有 5 種，然後 Python 又分 2.7 和 3.x，排列組合一下吧…

  • virtualenv
  • “native” pip
  • Docker
  • Anaconda
  • install from source

因為要來寫 ud730 的作業當作練習(天啊我超 lag)，作業是以 docker 的方式給的，所以一開始的安裝方式就會以 docker 的方式處理。稍微看了一下官方教學，所以順序大概會是

  • 裝 GPU 驅動
  • 裝 Docker CE(Community Edititon) for Fedora
  • 裝 nvidia-docker](https://github.com/NVIDIA/nvidia-docker
  • 跑 Hello World

裝 GPU 驅動

放進來篇幅太長，請參考我上一篇筆記在 Fedora 上安裝 Nvidia 1080Ti 的驅動

裝 Docker CE

可以參考官方教學，其實只有幾行，先處理一些前置作業，安裝 repo

然後安裝 docker

官方建議在 production 系統上應該要安裝同一版本的 docker-ce，所以安裝時要給版本參數

應該會看到歡迎訊息，如果不想用了要解除安裝

裝 nvidia-docker

官方教學在 nvidia-docker github 的 README.md 上

最後一行 $ nvidia-docker run --rm nvidia/cuda nvidia-smi 會跟 $ nvidia-smi 有一樣的輸出，就是裝 GPU driver 時曾經出現的這張

Hello Tensorflow

我們可以用類似下列的指令，從 tensor flow 的 repo 拉適合的 binary 來玩

測試是否安裝完成，validate your installation

進入 docker 之後，進入 python interactive shell
